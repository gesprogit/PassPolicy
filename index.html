🪟 Windows — Equipo standalone (sin Directorio Activo)

Aplica la política localmente mediante secedit.exe (Security Templates). No requiere Directorio Activo. Funciona en Windows 10/11 Pro y Server 2016+.

📄 password_policy_ENS.inf
Plantilla de seguridad Windows con todos los parámetros ENS configurados.
📜 apply_windows.bat
Script de aplicación. Clic derecho → "Ejecutar como administrador".
💙 apply_windows.ps1
Alternativa PowerShell con verificación y salida detallada.
Pasos de aplicación
  1. Descargar y extraer el ZIP en cualquier carpeta.
  2. Clic derecho sobre apply_windows.bat → Ejecutar como administrador.
  3. El script aplica la política y muestra confirmación.
  4. Para verificar: secpol.msc → Directivas de cuenta → Directiva de contraseñas.
Mecanismo: secedit /configure /areas SECURITYPOLICY · Efecto inmediato, sin reinicio.
🐧 Linux — Equipo standalone (PAM local)

Aplica la política mediante PAM (pam_pwquality, pam_faillock, login.defs). Compatible con Debian/Ubuntu, RHEL/CentOS/Fedora. No requiere servidor LDAP ni NIS.

📄 pwquality.conf
Complejidad y longitud de contraseña. Destino: /etc/security/pwquality.conf
📄 common-password
Módulo PAM: historial y hash SHA-512. Destino: /etc/pam.d/common-password
📄 common-auth
Bloqueo por intentos fallidos con pam_faillock. Destino: /etc/pam.d/common-auth
📄 login.defs.patch
Rotación y expiración de contraseñas. Fusionar en /etc/login.defs
🔧 apply_linux.sh
Script bash que detecta la distro, instala dependencias, hace backups automáticos y aplica todos los archivos en un solo paso.
Pasos de aplicación
  1. Descargar y extraer el ZIP.
  2. chmod +x apply_linux.sh
  3. sudo ./apply_linux.sh
  4. El script hace backups, instala dependencias y aplica la política.
  5. Para verificar: sudo chage -l <usuario>
Mecanismo: PAM (pam_pwquality + pam_faillock + pam_unix) + login.defs · Los backups se guardan automáticamente.
